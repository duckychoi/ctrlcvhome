# Reasoning Templates

## Chain of Thought 상세 템플릿

### 버그 디버깅용 CoT

```markdown
## 🔍 버그 분석: {{버그 제목}}

### 증상
- **현상**: {{발생하는 문제}}
- **재현 조건**: {{재현 방법}}
- **에러 메시지**: `{{에러 메시지}}`

### Step 1: 정보 수집
- 로그 확인: {{로그 내용}}
- 스택 트레이스: {{스택 트레이스}}
- 관련 코드: {{파일:라인}}

**중간 결론**: {{의심되는 영역}}

### Step 2: 가설 설정
- 가설 1: {{가설}}
- 가설 2: {{가설}}

**가장 유력한 가설**: {{선택된 가설}}

### Step 3: 가설 검증
{{검증 방법 및 결과}}

**결론**: {{확정된 원인}}

### Step 4: 해결
```{{language}}
// Before
{{문제 코드}}

// After
{{수정 코드}}
```

### Step 5: 검증
- [ ] 단위 테스트 통과
- [ ] 원래 에러 해결 확인
- [ ] 회귀 테스트 통과
```

## 추론 품질 체크리스트

### CoT 품질 확인

```markdown
## CoT 품질 검토

- [ ] 각 단계가 논리적으로 연결됨
- [ ] 중간 결론이 명확함
- [ ] 최종 결론이 단계들을 기반으로 도출됨
- [ ] 검증 가능한 결론임
- [ ] 누락된 단계가 없음
```

### ToT 품질 확인

```markdown
## ToT 품질 검토

- [ ] 모든 합리적인 옵션이 고려됨
- [ ] 평가 기준이 객관적임
- [ ] 가중치가 합리적임
- [ ] 각 옵션 평가가 공정함
- [ ] 결정 근거가 명확함
```

### ReAct 품질 확인

```markdown
## ReAct 품질 검토

- [ ] 각 사이클이 새로운 정보를 제공함
- [ ] Action이 Thought를 검증함
- [ ] Observation이 정확히 기록됨
- [ ] 무한 루프 방지됨 (최대 5 사이클)
- [ ] 결론이 관찰에 기반함
```
