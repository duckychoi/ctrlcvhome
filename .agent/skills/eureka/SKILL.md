# Eureka - 아이디어 발견 엔진

[EUREKA ACTIVATED - IDEA DISCOVERY ENGINE]

> "모든 위대한 아이디어는 단순한 한 문장에서 시작된다"

## 개요

**Eureka**는 사용자의 추상적 아이디어를 AI가 재귀적으로 탐색하고 발전시켜, 구체적인 MVP 제안으로 변환하는 스킬입니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   "여행 일정 공유"  ──────────────────────────────▶  3-4개     │
│   (추상적 아이디어)        AI 재귀적 사고           구체적 제안  │
│                                                                 │
│                              │                                  │
│                              ▼                                  │
│                     사용자 선택 후                              │
│              /screen-spec → /tasks-generator                    │
│                      자동 진행                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

## 사용법

```bash
/eureka 여행 일정 공유
/eureka 반려동물 건강 관리
/eureka 프리랜서 인보이스
/eureka 독서 모임 관리
/eureka 식단 기록
```

**입력**: 단어, 문장, 개념 (무엇이든 가능)

**출력**: 3-4개의 구체화된 MVP 제안

---

## 핵심 철학

### 1. 대부분의 아이디어는 이미 존재한다

> 새로운 것은 기존 것들의 새로운 조합이다.

- 여행 + 협업 = TripIt, Wanderlog
- 건강 + 기록 = MyFitnessPal, 삼성헬스
- 인보이스 + 자동화 = FreshBooks, Wave

AI는 이 패턴을 알고 있으므로, 빠르게 구체화할 수 있습니다.

### 2. 재귀적 사고로 진화

```
Round 1: 이건 뭐지? → 핵심 개념 추출
Round 2: 비슷한 게 뭐가 있지? → 기존 솔루션 조사
Round 3: 뭐가 부족하지? → 한계점 분석
Round 4: 어떻게 다르게 할 수 있지? → 차별화 도출
Round 5: 가장 간단하게 만들면? → MVP 정의
```

### 3. 선택의 힘

3-4개 제안 중 사용자가 선택함으로써:
- 방향성 확정
- 즉시 실행 가능
- 소크라테스 21개 질문 생략

## 워크플로우

### Phase 1: 의미 분석 (Semantic Analysis)

사용자 입력에서 핵심 요소 추출:

```
입력: "여행 일정 공유"

추출:
├── 도메인: 여행 (Travel)
├── 핵심 기능: 일정 관리, 공유
├── 타겟 사용자: 여행자, 그룹
├── 암묵적 니즈: 협업, 편의성
└── 키워드: travel, itinerary, share, collaborate
```

### Phase 2: 재귀적 탐색 (Recursive Exploration)

**3-5 라운드 내부 사고** (사용자에게 보이지 않음):

```
[내부 사고 Round 1/5]
질문: "여행 일정 공유"의 본질은 무엇인가?
탐색: 여행 계획 → 일정 → 공유 → 협업 → 그룹 의사결정
결론: 핵심은 "그룹 여행 협업"

[내부 사고 Round 2/5]
질문: 기존에 어떤 솔루션이 있는가?
탐색: WebSearch("travel itinerary sharing app 2024")
발견: TripIt, Wanderlog, Google Travel, Sygic
분석: 대부분 개인 중심, 실시간 협업 부족

[내부 사고 Round 3/5]
질문: 기존 솔루션의 한계는?
탐색: 리뷰, 비교 기사 분석
발견:
  - 실시간 동시 편집 없음
  - 비용 분담 기능 부족
  - 현지인 추천 연동 없음
  - 오프라인 지원 미흡

[내부 사고 Round 4/5]
질문: 차별화할 수 있는 방향은?
탐색: 한계점 × 기술 가능성
도출:
  - A: 실시간 협업 + 투표
  - B: 현지인 매칭 + AI 추천
  - C: 초간단 링크 공유 (노앱)
  - D: 오프라인 우선 + 동기화

[내부 사고 Round 5/5]
질문: 가장 간단한 MVP는?
평가: 복잡도, 개발 기간, 가치
결론: C (링크 공유)가 가장 간단하면서 가치 있음
```

## 제안 카드 구조

각 제안은 다음 정보를 포함:

| 항목 | 설명 |
|------|------|
| **이름** | 서비스명 (영문 + 한글 설명) |
| **가치 제안** | 한 문장으로 핵심 가치 |
| **주요 기능** | 3개의 핵심 기능 |
| **타겟 사용자** | 누구를 위한 서비스인지 |
| **차별점** | 기존 솔루션 대비 차이 |
| **예상 화면 수** | MVP 기준 화면 개수 |
| **복잡도** | 간단/보통/복잡 (●○○○○ ~ ●●●●●) |

### MVP 추천 기준

```
복잡도 점수 = 화면 수 × 기능 복잡도 × 외부 연동 수

점수 1-10: 간단 (⭐ MVP 추천)
점수 11-20: 보통
점수 21+: 복잡
```

## /socrates와의 차이

| 항목 | /socrates | /eureka |
|------|-----------|---------|
| **입력** | 대화형 21개 질문 | 단어/문장 하나 |
| **과정** | 사용자와 대화 | AI 내부 재귀 사고 |
| **시간** | 30분-1시간 | 1-2분 |
| **출력** | 6개 기획 문서 | 3-4개 제안 카드 |
| **적합 상황** | 복잡한 서비스, 명확한 비전 | 빠른 검증, MVP |

### 언제 뭘 쓸까?

```
"아이디어가 막연해요" → /eureka (빠른 구체화)
"방향은 정해졌어요" → /socrates (심층 기획)
"그냥 빨리 만들고 싶어요" → /eureka (즉시 개발)
```

## 옵션

```bash
/eureka 아이디어           # 기본 (3-4개 제안)
/eureka 아이디어 --deep    # 심층 분석 (5-6개 제안, 더 많은 라운드)
/eureka 아이디어 --fast    # 빠른 분석 (2개 제안, 최소 라운드)
/eureka 아이디어 --ko      # 한국어 서비스명 우선
/eureka 아이디어 --en      # 영어 서비스명 우선
```

## 출력 후 자동 진행

사용자 선택 후 자동으로:

1. **선택된 제안 → 임시 기획 문서 생성**
   ```
   .claude/docs/eureka-proposal.md
   ```

2. **/screen-spec 실행**
   - eureka-proposal.md 기반 화면 명세 생성
   - 자동 모드로 빠르게 진행

3. **/tasks-generator 실행**
   - 화면 명세 기반 태스크 생성
   - TASKS.md 자동 생성

4. **개발 준비 완료 알림**
   ```
   ✅ Eureka → Screen Spec → Tasks 완료!

   📁 생성된 파일:
   ├── .claude/docs/eureka-proposal.md
   ├── specs/screens/*.yaml
   └── TASKS.md

   🚀 다음 단계: /auto-orchestrate 또는 직접 개발 시작
   ```
