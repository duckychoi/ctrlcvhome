# 질문 출발점 및 탐색 영역 (v2.0)

> **주의: 이 파일은 고정된 질문 목록이 아닙니다.**
>
> 질문은 사용자 답변에 따라 실시간으로 생성됩니다.
> 이 파일은 탐색 영역과 출발점만 제공합니다.

---

## 대화 시작 (유일한 고정 질문)

모든 대화는 이 하나의 질문으로 시작합니다:

```
"무엇을 만들고 싶으세요?"
```

이 질문에 대한 답변을 분석하여 모든 후속 질문이 동적으로 생성됩니다.

## 필수 탐색 영역

### 1. 욕망 (Desire) - 왜?

**목표:** 표면적 욕구 → 진짜 욕구 → 숨겨진 두려움

**출발 질문들 (상황에 맞게 선택):**
```
"왜 이걸 만들고 싶어요?"
"이걸 만들면 뭐가 달라지나요?"
"성공하면 어떤 기분일 것 같아요?"
```

**심층 탐색 (5 Whys):**
```
답변: "돈을 벌고 싶어서요"
→ "왜 돈을 벌고 싶어요?"
→ "왜 그게 중요한가요?"
→ "그게 없으면 어떻게 되나요?"
→ "그 두려움의 정체는 뭔가요?"
```

### 2. 페르소나 (Who) - 누구?

**목표:** "모든 사람" → 구체적 세그먼트

**출발 질문들:**
```
"누가 이걸 쓸까요?"
"그 사람은 지금 뭘 하고 있나요?"
"그 사람의 하루를 상상해볼 수 있어요?"
```

**도발 질문 (과잉 일반화 시):**
```
"모든 사람이라고요? 정말요?
 당신의 할머니도 쓸까요?
 10살짜리 아이도요?"
```

### 3. 문제 (Problem) - 어떤 고통?

**목표:** 현상 → 근본 원인

**출발 질문들:**
```
"지금 뭐가 불편해요?"
"그 불편함이 언제 가장 심해요?"
"그 문제가 해결되면 뭐가 달라지나요?"
```

**비교 질문:**
```
"기존에 쓰던 건 뭐였어요?"
"그게 왜 마음에 안 들었어요?"
"있는 것 중에 비슷한 건 없나요?"
```

### 4. 핵심 기능 (What) - 무엇이 필수?

**목표:** 욕심 목록 → MVP 1~3개

**출발 질문들:**
```
"꼭 있어야 할 기능이 뭐예요?"
"단 하나만 있다면 뭘 선택할래요?"
"없으면 앱이 의미 없는 기능은요?"
```

**모순 탐지 질문:**
```
"앞서 '심플한 앱'이라고 하셨는데,
 지금 10개 기능을 말씀하고 계세요.
 뭐가 진짜 중요한 건가요?"
```

### 5. 성공 기준 (Success) - 어떻게 측정?

**목표:** 막연한 성공 → 측정 가능한 지표

**출발 질문들:**
```
"이게 성공했다는 걸 어떻게 알 수 있어요?"
"1년 후 성공했다면, 어떤 숫자가 보일까요?"
"실패했다면 뭐가 잘못된 걸까요?"
```

**레벨별 질문:**
- L1: "사용자가 기뻐하면 성공인가요?"
- L4: "DAU, Retention, LTV 목표가 뭔가요?"

## 선택 탐색 영역 (레벨에 따라)

### 6. 두려움 (Fear) - L2+

```
"이 프로젝트가 실패하면 어떻게 되나요?"
"가장 피하고 싶은 결과는 뭔가요?"
"밤에 잠들기 전 가장 걱정되는 건 뭔가요?"
```

### 7. 차별화 (Why You) - L3+

```
"왜 당신이 이걸 만들어야 하나요?"
"다른 사람이 만들면 안 되나요?"
"경쟁자와 뭐가 다른가요?"
```

### 8. 리스크 (Risk) - L3+

```
"이 프로젝트의 가장 큰 위험은 뭔가요?"
"실패할 가능성이 가장 높은 시나리오는요?"
"그 위험을 줄이려면 뭘 해야 하나요?"
```

### 9. 기술 (Tech) - L3+

```
"기술적으로 가장 어려운 부분은 뭔가요?"
"어떤 기술로 만들고 싶어요?"
"확장성은 얼마나 중요한가요?"
```

### 10. 비즈니스 (Business) - L4

```
"돈은 어떻게 벌 계획이에요?"
"고객 획득 비용(CAC)은 얼마나 들 것 같아요?"
"손익분기점은 언제쯤일까요?"
```

## 슬랙 웹훅 설정 (선택 사항)

대화 시작 전 슬랙 알림 설정을 물어봅니다:

```json
{
  "questions": [{
    "question": "기획 진행 상황을 슬랙으로 알림 받으시겠어요?\n\n(선택 사항 - 필요 없으시면 '건너뛰기'를 선택하세요)",
    "header": "슬랙 알림",
    "options": [
      {"label": "건너뛰기", "description": "슬랙 알림 없이 진행"},
      {"label": "웹훅 URL 입력", "description": "Other를 선택하여 URL 직접 입력"}
    ],
    "multiSelect": false
  }]
}
```

**⚠️ 중요:** 하드코딩된 웹훅 URL을 절대 사용하지 마세요!

## 기술 스택 질문 (대화 후반) - 레벨별 차등 적용!

> **핵심:** 사용자 레벨에 따라 질문 방식이 달라집니다.
> 상세 규칙: `tech-stack-recommendation.md` 참조

필수 탐색 영역이 완료된 후, **레벨에 따라** 기술 스택을 결정합니다:

### 레벨별 전략 요약

| 레벨 | 전략 | 설명 |
|------|------|------|
| **L1** | 자동 제안 | AI가 선택, 확인만 요청 |
| **L2** | 추천 + Yes/No | AI 추천 + 간단한 동의 |
| **L3** | 선택지 제공 | 3-4개 옵션 + 장단점 |
| **L4** | 트레이드오프 | 질문으로 시작, 자유 선택 |

### L1/L2: 자동 제안 (결정 부담 최소화)

```json
{
  "questions": [{
    "question": "기술적인 부분은 제가 정리해드릴게요.\n\n당신의 프로젝트에 가장 적합한 기술 조합:\n\n📦 프론트엔드: React + Vite\n🔧 백엔드: FastAPI (Python)\n💾 데이터베이스: PostgreSQL\n\n이대로 진행할까요?",
    "header": "기술 스택",
    "options": [
      {"label": "네, 좋아요!", "description": "추천대로 진행"},
      {"label": "잘 모르겠어요", "description": "좀 더 설명해주세요"}
    ],
    "multiSelect": false
  }]
}
```

### L3: 선택지 + 장단점

```json
{
  "questions": [{
    "question": "백엔드 프레임워크를 선택해주세요.\n\n| 옵션 | 장점 | 단점 |\n|------|------|------|\n| FastAPI | 성능, 타입, 문서 자동화 | 생태계 작음 |\n| Django | 풀스택, ORM, Admin | 무거움 |\n| Express | 유연함, Node 통일 | 구조 자유도 |\n\n[AI 추천: FastAPI]",
    "header": "백엔드",
    "options": [
      {"label": "FastAPI", "description": "추천 - Python"},
      {"label": "Django", "description": "풀스택 Python"},
      {"label": "Express", "description": "Node.js"},
      {"label": "NestJS", "description": "Node.js 구조화"}
    ],
    "multiSelect": false
  }]
}
```

### L4: 트레이드오프 질문

```json
{
  "questions": [{
    "question": "기술 스택은 어떻게 생각하세요?\n\n몇 가지 고려 포인트:\n\n1️⃣ SSR이 필요한가요, CSR로 충분한가요?\n2️⃣ Python vs Node.js 중 팀 역량은요?\n3️⃣ 초기 개발 속도 vs 장기 유지보수, 뭐가 우선?\n\n선호하는 스택이 있으시면 말씀해주세요.",
    "header": "기술 스택",
    "options": [
      {"label": "이미 정해놨어요", "description": "Other로 직접 입력"},
      {"label": "논의해보고 싶어요", "description": "트레이드오프 분석"},
      {"label": "추천해주세요", "description": "제 상황에 맞는 추천"}
    ],
    "multiSelect": false
  }]
}
```

### 프로젝트 유형별 자동 감지

| 키워드 | 추천 변경 |
|--------|----------|
| "SEO", "검색 노출" | Next.js 추천 |
| "실시간", "채팅" | WebSocket + Redis |
| "AI", "ML" | FastAPI + Vector DB |
| "빠르게", "MVP" | Supabase 권장 |

## 🎯 화면 기획 질문 (Phase 2 - 핵심!)

> 핵심 기능이 도출되면, 반드시 화면 매핑 질문으로 넘어갑니다.
> 상세 가이드: `feature-screen-mapping.md` 참조

### 화면 존재 확인

```
"[기능]을 하려면 어떤 화면이 필요할까요?"
"그 화면에 들어가면 뭐가 제일 먼저 보이나요?"
"몇 개 화면이 필요할 것 같아요?"
```

### 화면 구성 요소

```
"그 화면에 뭐가 있나요? 버튼? 입력창? 목록?"
"목록이면 한 항목에 뭐가 보이나요?"
"버튼을 누르면 뭐가 일어나나요?"
```

### 화면 간 연결 (집요하게!)

```
"이 화면에 어떻게 들어오나요?"
"[버튼]을 누르면 어디로 가나요?"
"저장 성공하면 어디로 가나요?"
"뒤로 가기 누르면요?"
```

### 예외 상황

```
"데이터가 없으면 뭐가 보이나요?"
"에러나면 어떻게 알려주나요?"
"로그인 안 했으면요?"
```

### 화면 목록 확인

```json
{
  "questions": [{
    "question": "지금까지 나온 화면을 정리해볼게요:\n\n1. 홈 화면\n2. 입력 화면\n3. 목록 화면\n4. 상세 화면\n\n이 화면들이 맞나요? 빠진 게 있나요?",
    "header": "화면 확인",
    "options": [
      {"label": "네, 맞아요", "description": "다음으로 진행"},
      {"label": "빠진 화면 있어요", "description": "추가할 화면 입력"},
      {"label": "필요 없는 화면 있어요", "description": "제거할 화면 지정"}
    ],
    "multiSelect": false
  }]
}
```

## 중요 규칙 (다시 강조)

### ❌ 하지 말 것

1. **Q1→Q2→Q3 순서대로 질문하지 마세요**
2. **같은 질문을 모든 사용자에게 하지 마세요**
3. **답변을 무시하고 다음으로 넘어가지 마세요**
4. **"좋은 생각이에요"라고 빈말하지 마세요**

### ✅ 반드시 할 것

1. **사용자 답변의 키워드를 인용하세요**
2. **"왜?"를 물어서 진짜 동기를 파악하세요**
3. **모순을 발견하면 지적하세요**
4. **레벨에 맞는 언어를 사용하세요**

## 세렌디피티 기능 (핵심 기능 탐색 후)

핵심 기능 영역 탐색 완료 후, 창의적 연결을 제안합니다:

**8가지 연결 도메인:**
1. 역사 - 과거의 유사 사례
2. 산업 - 다른 산업의 성공 모델
3. 인문학 - 인간 본성, 철학
4. 트렌드 - 최신 기술/사회 트렌드
5. 과학 - 과학적 원리, 자연 영감
6. 예술 - 미학, 감각적 경험
7. 문학 - 스토리텔링, 서사 구조
8. 음악 - 리듬, 조화, 감정적 공명
